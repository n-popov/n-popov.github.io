Циклы

В C++ существует три варианта цикла, но использовать мы будем только один:
for(/* инициализация */;/* условие */;/* инкремент / декремент */) {
    /* тело цикла */
}
 Любые из прокомментированных блоков могут быть пустыми. Благодаря этой возможности цикл for делает полностью ненужным цикл while, а в комбинации с условным оператором и break — и do (...) while. Кстати, в языке Go компании Google действительно нет других циклов, кроме for.

Работа с памятью в C++

Ячейки памяти

Переменные в языке C++ хранятся в ячейках вместимостью в 1 байт. Таким образом, переменная типа int, например, занимает 4 ячейки, а переменная типа double — 8. У каждой ячейки есть свой адрес — шестнадцатеричное число. Аналогично тому, как мы можем назвать адрес своего дома, у переменной можно узнать адрес первой из занимаемых ею ячеек. Для этого необходимо воспользоваться оператором & взятия адреса. Адрес переменной можно положить в другую переменную специального типа: указатель на переменную. По указателю можно получить исходную переменную с помощью обратного оператора — оператора * разыменования. К указателям можно прибавлять целые числа, «прыгая» по ячейкам памяти вперёд и назад. Однако, длина прыжка зависит от типа указателя (зависящего от переменной, на которую он указывает). Например, указатель на int — int* — при прибавлении к нему единицы на большинстве машин станет указывать на 4 ячейки правее.

Структура памяти процесса

Что такое процесс? Можно сказать, это процесс выполнения некоторой задачи. Представьте, что вы делаете физлабы. Для успешной процедуры выполнения необходимы исполнитель (вы), оборудование и кабинет. Аналогично дела обстоят с приложениями на компьютере. Для успешного выполнения процесса необходим исполнитель (процессор) и место в памяти. Под каждый процесс выделен свой участок памяти. Память внутри него структурирована особым образом: часть памяти выделяется под код программы, часть — под локальные переменные (т.н. стек — действительно реализует принцип LIFO, но исключительно в плане расположения переменных: естественно, свободный доступ предоставляется к любой локальной переменной).

Массивы

Массивы бывают статические и динамические. Например, list в Python является динамическим, то есть его размер может изменяться по ходу выполнения программы, а array в Go — статическим — его размер фиксируется во время компиляции.

В C++, внезапно, массив — это указатель. Указатель на первую ячейку памяти, где лежат элементы массива (то есть на первый элемент массива) — ведь, помня про арифметику указателей, мы можем добраться и до всех остальных. Статические массивы лежат на стеке; память под динамические же процесс запрашивает напрямую у ОС в оперативной памяти во время выполнения программы.
