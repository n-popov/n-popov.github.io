Упражнения:

  1. (50 б.) Второй максимальный элемент

    В наборе из T последовательностей размера N для каждой последовательности найдите второй максимальный элемент для каждой смежной пересекающейся (см. пример) подпоследовательности размера K (K ≤ N).

    Формат ввода

    Нулевой строкой вводится размер набора T. На 2i + 1 строке, i ∈ [0 .. T] вводятся два числа Ni и Ki для следующей последовательности, а на 2i + 2 строке — сама последовательность размера Ni.

    Формат вывода

    Т строк с наборами искомых элементов для каждой последовательности.

    Пример
    Ввод
    2
    5 2
    3 4 6 3 4
    7 4
    3 4 5 8 1 4 10
    Вывод
    3 4 3 3
    5 5 5 8

    Пояснение: для первой последовательности имеем Ni = 5, Ki = 2, подпоследовательности — это {3, 4}, {4, 6}, {6, 3} {3, 4}. Для второй Ni = 7, Ki = 4, подпоследовательности — это {3, 4, 5, 8}, {4, 5, 8, 1}, {5, 8, 1, 4}, {8, 1, 4, 10}.

  2. (50 б.) Классические исключения и дробь
    Реализуйте простой класс для хранения рациональных чисел в виде дроби с целыми числителем и
знаменателем. Реализуйте для данного класса базовые операторы для действий с рациональными числами.

Прототип класса:

class Rational {
public:
    // Конструктор дроби, здесь a - числитель, b - знаменатель
    Rational(int a, int b);

    // Реализуйте операторы:
    // - сложения двух дробей
    // - вычитания двух дробей
    // - умножения двух дробей
    // - деления двух дробей
    // - умножения дроби на целое число (должно работать при любом порядке операндов)
};
Также реализуйте оператор вывода, который должен выводить дробь в формате: a/b, и оператор ввода,
который должен получить дробь из формата a/b. Если дробь введена в нужном формате
«числосимволчисло»,
но при этом символ не является знаком деления, выбросите исключение с сообщением о том, что формат
данных некорректен.
Разработайте своё исключение или воспользуйтесь одним из стандартных.

В случае деления на 0 (в операторе, конструкторе или вводе при b = 0) выбросите другое исключение
с сообщением о том, что делить на 0 запрещено.

Также реализуйте сокращение дроби и оператор приведения к double.
